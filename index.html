Towards Resolving the API Selection Dilemma in Market-scale Android Malware Detection

Background

All the functions of an Android app are empowered by the Java framework APIs offered by Android SDK.Hence, studying the framework APIs employed by an app has long been considered a natural approach to determining the app’s malice. In this project, we explore the practical feasibility of using framework APIs to detect malware in a large-scale app market, and the detection time should be short enough.

API Set

We study ground-truth data from one of the world’s largest Android app markets (called Market-T), and extract 426 key APIs (out of 50K APIs) which generally belong to the following three categories (with some overlap). The 426 key APIs (out of 50K APIs) are of fundamental importance to malware detection.

  • Restricted API:112 framework APIs whose usage should be restricted with dangerous-level permissions
