Engineering API-based Android Malware Detection at Market Scales

Background

This work introduces the design, implementation, measurement findings, and deployment experiences of APIChecker, a system that performs automated, accurate, and fast malware detection for Android markets. APIChecker has been deployed at one of the world’s largest Android app stores since 2018, vetting around 10K newly submitted app every day. It executes submitted apps in a sandbox, monitors their framework API invocations, and employs machine learning to identify malware using the API call logs. We take a principled, big-data-driven approach for identifying the appropriate set of APIs to be monitored, and carefully engineer the system to ensure its high accuracy, performance, and reliability. During its commercial deployment, APIChecker achieves an overall precision of 98% and recall of 96%, with an average per-app scan time of 1.3 minutes. Our work demonstrates the feasibility of using API-centric dynamic analysis, enhanced by judicious feature selection and system-level optimization, to develop an app-vetting system that yields very high accuracy and short scanning time for a large commercial Android app market. We will release our measurement data to the research community.

API Set

We study ground-truth data from one of the world’s largest Android app markets (called Market-X), and extract 426 key APIs (out of 50K APIs) which generally belong to the following three categories (with some overlap). The 426 key APIs (out of 50K APIs) are of fundamental importance to malware detection.

  • Highly-Correlation API: 260 framework APIs that have significant statistical correlations (|PCC| > 0.2) with the malice of apps.
  • Restricted API: 112 framework APIs whose usage should be restricted with dangerous-level permissions and those signature-level permissions with certain privileges.
  • Sensitive Operation API: 70 framework APIs that can enable sensitive operations for conducting certain attacks (e.g., ransomware and background malware).
